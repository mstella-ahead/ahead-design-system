## Recipe — building a new page

A typical AHEAD page combines patterns in this order:

**Landing page**: Hero (F) → Eyebrow+Display+Body (A) ×3–5 → Stats (C) → Partner grid (D) → Numbered services (B) → Final CTA (A)

**Article page**: Article hero (G) → Two-column body (H) interleaved with pull-quotes (I) → Related (L)

**Case study page**: Article hero (G) → Overview (A) → Listicle-image (J) ×3–6 → Testimonial quote (I) → Related cases (L). Sticky share sidebar (K) runs throughout.

See `../docs/patterns.md` for the full pattern library.
